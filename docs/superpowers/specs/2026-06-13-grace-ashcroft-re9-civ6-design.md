# 文明六 Mod 设计稿：厄尔庇斯协议 - 格蕾丝·阿什克罗夫特

## 概述

本 Mod 以“生化危机9：格蕾丝”为主题，为《文明 VI》加入一个偏科研、清剿、样本转化玩法的新文明与领袖。

核心循环是：

**清理蛮族威胁 -> 获得感染者血液 -> 通过罗兹山疗养院项目进行处理 -> 转化为战斗强化、科研收益或尤里卡。**

这个文明不应像传统扩张型国家，而更像一套灾害应对与生物安全协议。玩家通过主动侦察、清剿蛮族、压制地图上的失控威胁，把战斗中回收的高危样本转化为制度化的科研与作战优势。

## 核心身份

### 文明：厄尔庇斯协议

**定位：** 生物安全研究体制  
**玩法标签：** 反蛮族、战斗清图、学院项目、科研转化  
**核心节奏：** 前线击杀获取资源，后方城市项目消化资源，再反哺军事与科技。

### 领袖：格蕾丝·阿什克罗夫特

**定位：** 事件幸存者、现场调查者、隔离区行动核心  
**玩法标签：** 把感染事件转化为证据、药剂、战术优势与科研进展  
**推荐打法：** 主动找蛮族、尽早解锁学院、围绕罗兹山疗养院持续跑项目。

## 文明能力：厄尔庇斯协议

研究 **“文字”** 并解锁 **学院** 后，文明获得 **“采血器”**。

采血器启用后，玩家击杀蛮族单位会获得特殊资源 **“感染者血液”**。感染者血液不是地图上的战略、奢侈或加成资源，而是一个隐藏的玩家库存，用于支付罗兹山疗养院解锁的特殊项目。

### 建议规则文本

研究“文字”并解锁学院后，获得“采血器”。此后击杀蛮族单位可获得“感染者血液”。拥有罗兹山疗养院项目的城市可以消耗感染者血液，将其转化为战斗强化、大科学家点数、科技值或尤里卡。

## 特色建筑：罗兹山疗养院

**建筑类型：** 学院体系特色建筑，具体替代图书馆、大学、研究实验室，还是作为独立学院建筑，后续实现时再定。  
**主题：** 集隔离医院、样本库、病理实验室于一体的高危研究设施。

### 建筑效果

罗兹山疗养院建成后，所有城市追加一组特殊项目。所有特殊项目均消耗 **10 生产力**，并在完成时消耗感染者血液。

“首都可以额外建造内容”建议先整理为以下设计意图：

- 首都更稳定地获得罗兹山疗养院或其前置能力。
- 第一座罗兹山疗养院作为文明项目体系的中央解锁点。
- 第一座罗兹山疗养院建成后，所有城市都能执行感染者血液相关项目。

### 建议规则文本

特色学院建筑。建成后，所有城市解锁感染者血液制备与研究项目。这些项目消耗少量生产力，并将感染者血液转化为军事强化或科研收益。

## 特殊资源：感染者血液

**类型：** 玩家隐藏库存  
**来源：** 采血器启用后击杀蛮族单位  
**用途：** 支付罗兹山疗养院特殊项目  
**设计目的：** 把清剿蛮族的前线行动和学院科研体系绑定起来。

### 获取规则

研究“文字”并解锁学院后，击杀蛮族单位获得感染者血液。

初始调校建议：

- 击杀普通蛮族单位：+1 感染者血液。
- 清理蛮族营地或击杀高强度蛮族单位，后续可以考虑追加奖励，用于改善中后期节奏。

## 特殊项目：制备项目

所有制备项目都由罗兹山疗养院体系解锁，基础成本为 **10 生产力**。这些项目应该是短周期、可重复的实验室处理流程，真正限制玩家的是感染者血液库存，而不是生产力。

### 1. 溶血剂制备

**主题：** 针对感染样本研发的蛮族压制试剂。  
**成本：** 10 生产力，消耗感染者血液。  
**奖励：** 获得“溶血剂”强化。

**效果：** 所有单位对蛮族单位 **+5 战斗力**。

**建议：** 如果允许重复获取，必须设置上限。无上限的对蛮族 +5 很容易让早期清图失衡。第一版建议做成“最多 3 次”，或更保守地做成“一次性永久强化”。

### 2. 稳定剂制备

**主题：** 让战斗人员适应高危环境的稳定化处理。  
**成本：** 10 生产力，消耗感染者血液。  
**奖励：** 获得“稳定剂”强化。

**效果：** 所有战斗单位 **+5 战斗力**。最多完成 **3 次**。

**最大效果：** 所有战斗单位 +15 战斗力。

### 3. 类固醇制备

**主题：** 应急强化与再生治疗方案。  
**成本：** 10 生产力，消耗感染者血液。  
**奖励：** 获得“类固醇”强化。

**效果：** 战斗单位防御时额外 **+5 战斗力**，并且每回合自动回复 **5 生命值**。最多完成 **3 次**。

**最大效果：** 防御时 +15 战斗力，每回合 +15 生命值。

## 特殊项目：科研转化项目

科研转化项目把感染者血液转化为科技进度、伟人点数或尤里卡。它们不直接提高军队强度，而是让清剿行动成为文明科技发展的燃料。

### 4. 血液样本分析

**主题：** 对感染样本进行快速筛查，提取当前时代可用的技术线索。  
**成本：** 10 生产力，消耗 **3 感染者血液**。  
**奖励：** 随机触发一个当前时代可获得的尤里卡。

如果当前没有可触发的尤里卡，则改为获得科技值。

### 科技值补偿

初始调校建议：

- 远古/古典基准：100 科技值。
- 后续每个时代递增。

建议第一版公式：

`100 + 50 * 当前时代序号`

### 5. 异常病理研究

**主题：** 对异常感染路径进行正式病理研究，转化为科研人才招募优势。  
**成本：** 10 生产力，消耗 **3 感染者血液**。  
**奖励：** 大科学家点数。

初始调校建议：

- 基础奖励：+30 大科学家点数。
- 每个时代额外 +10 大科学家点数。

建议第一版公式：

`30 + 10 * 当前时代序号`

## 第三个科研项目建议

你设定里提到“做三种项目”，但目前科研转化项目只明确了两个。建议补一个中间型项目，避免“全靠尤里卡”和“全靠大科学家点数”两条线太窄。

### 6. 隔离协议复盘

**主题：** 将战斗报告、感染样本、城市隔离流程整理成可复用的科研制度。  
**成本：** 10 生产力，消耗 **3 感染者血液**。  
**奖励：** 获得科技值，并获得少量大科学家点数。

初始调校建议：

- 科技值：`50 + 25 * 当前时代序号`
- 大科学家点数：`10 + 5 * 当前时代序号`

这个项目作为稳定收益项，强度低于血液样本分析的尤里卡上限，也低于异常病理研究的大科学家点数专精。

## 平衡注意

这套设计最大的风险是永久战斗力叠加。稳定剂与类固醇如果都全局生效，并且各叠三次，会让中期战斗强度非常高。第一版建议使用以下限制：

- 溶血剂：一次性 +5 对蛮族，或最多叠加 3 次。
- 稳定剂：最多叠加 3 次。
- 类固醇：最多叠加 3 次。
- 每个制备项目至少消耗 3 感染者血液，除非测试后发现血液收入太慢。

10 生产力的项目成本可以保留。它让项目像“实验室处理流程”，而真正的成本来自外勤击杀和血液库存管理。

## 实现方向

参考样例“东方project：妖精领袖包”，这套 Mod 可以按以下层次实现：

- 用 SQL/XML 定义文明、领袖、特色建筑、项目、文本、图标。
- 用 Lua 把感染者血液记录为玩家属性。
- 监听单位击杀事件，判断被击杀单位是否属于蛮族阵营。
- 在项目完成时消耗感染者血液，并发放对应奖励。
- 能用 Modifier 表达的永久战斗力加成，优先用 Modifier。
- 需要重复、计数、上限、随机尤里卡的奖励，用 Lua 处理。

第一版建议不要做自定义 UI。先用项目文本说明感染者血液消耗，并把库存显示放到日志或简易提示中。等核心循环跑通后，再追加一个显示当前感染者血液库存的小 UI 面板。

| 感染者血液作为真实资源的替代方案 | SQL/XML + `PlayerResources` Lua | 如果以后想让游戏原生资源系统处理库存，可定义 `RESOURCE_INFECTED_BLOOD`，再用 `PlayerResources:ChangeResourceAmount` 增减。这样更容易查数量、可复用资源接口，但要处理显示、交易、库存上限和其他文明可见性。 |
| 击杀蛮族获得血液 | Gameplay Lua | 监听战斗/击杀事件，判断被击杀玩家是否蛮族、击杀者是否格蕾丝文明、采血器是否启用，然后增加感染者血液。具体事件名需要实机验证；资料中可查 `Events.Combat`，也有资料提到 `Events.UnitKilledInCombat`。 |
| 清理蛮族营地奖励血液 | Gameplay Lua | 如果要做营地额外奖励，需要监听营地清除、改良移除或相关事件；第一版可以先不做，只做击杀奖励，降低事件兼容风险。 |
| 罗兹山疗养院 | SQL/XML + Modifier，必要时 Lua | 数据库定义特色学院建筑。若“建成第一座后所有城市解锁项目”可用数据库 Requirement/Modifier 表达则优先数据库；若表达困难，改由 Lua 在建筑完成事件中给玩家写属性，再由项目完成逻辑读取。 |
| 首都额外建造内容 | SQL/XML Requirement 或 Lua | 如果只是“首都可建特殊建筑/项目”，优先用数据库要求限制城市为首都。如果是“首都建成后全局解锁”，更适合 Lua 设置玩家属性。 |
| 特殊项目本体 | SQL/XML | 在 `Projects` 相关表中定义项目、成本、文本、前置条件。10 生产力成本可以直接放数据库。 |
| 项目消耗感染者血液 | Gameplay Lua，或真实资源方案用数据库资源成本 | 如果感染者血液用 Player Property，项目完成时 Lua 检查并扣除。若血液不足，第一版建议不给奖励并弹通知；更好的体验是后续加 UI/可用性限制。如果血液做成真实资源，则可以尝试使用项目资源成本表交给数据库处理。 |
| 溶血剂：对蛮族 +5 | Modifier + Lua 计数 | 战斗力效果本身用 Modifier/UnitAbility。项目完成后 Lua 增加层数并 `AttachModifierByID` 或切换对应等级 Modifier。第一版建议最多 1-3 层，避免无限叠加。 |
| 稳定剂：全战斗单位 +5，最多三次 | Modifier + Lua 计数 | 用三个等级 Modifier 最稳：稳定剂 I/II/III 每层 +5。Lua 负责项目完成时检查上限、扣血液、挂下一层 Modifier。 |
| 类固醇：防御 +5、每回合回复 5，最多三次 | Modifier + Lua 计数，回血需实测 | 防御战斗力优先用 Modifier/Requirement 表达；每回合回血如果找得到合适 Modifier 就用 Modifier，否则用 Lua 在回合事件里遍历符合条件单位并修改伤害。 |
| 血液样本分析：随机尤里卡 | Gameplay Lua | 需要遍历当前时代科技，排除已研究、已触发尤里卡、不能触发的科技，再用 `PlayerTechs:TriggerBoost` 触发。无可用尤里卡时走科技值补偿。 |
| 科技值补偿 | Gameplay Lua | 使用 `PlayerTechs:ChangeCurrentResearchProgress` 给当前研究增加进度，或实测 `Player:GrantYield(YIELD_SCIENCE)` 是否符合预期。数值从 `GlobalParameters` 读取，方便超级版覆盖。 |
| 异常病理研究：大科学家点数 | Gameplay Lua 或 Modifier | 简单做法是项目完成后用 `PlayerGreatPeoplePoints:ChangePointsTotal` 加大科学家点数。也可以用一次性 Modifier，但 Lua 更方便做时代缩放。 |
| 隔离协议复盘：科技 + 大科学家点数 | Gameplay Lua | 复用科技值和伟人点数发放函数，作为稳定收益项目。 |
| 时代缩放 | Lua + `GlobalParameters` | Lua 读取玩家/游戏时代，再套公式。基准值放进 `GlobalParameters`，以后普通版/超级版只改 SQL 参数，不复制 Lua。 |
| 强化层数上限 | Lua + Player Property | 用玩家属性记录 `GRACE_STABILIZER_LEVEL`、`GRACE_STEROID_LEVEL`、`GRACE_HEMOLYTIC_LEVEL`。项目完成时检查上限。 |
| 当前血液数量显示 | UI Lua + UI XML，后置 | 第一版可以不做。后续若做 UI，读 `Player:GetProperty("INFECTED_BLOOD")` 或真实资源数量，在顶部栏、报告面板或项目提示中显示。 |
| 通知与反馈 | Gameplay Lua + UI 事件 | 项目成功、血液不足、尤里卡失败转科技值等都应发通知或日志。先用简单通知，后续再做专门 UI。 |
| 文明百科、项目说明、按钮文本 | `UpdateText` SQL/XML | 全部本地化文本放 Text 文件，后续支持中英双语时也好维护。 |
| 图标、领袖图、项目图 | `UpdateIcons` + BLP/ArtDefs | 第一版用占位图标即可。自定义领袖图、项目图、建筑图标后续再进 ArtDefs/BLP 资源流程。 |
| 普通版/超级版 | 主 Mod + Add-on Mod | 主 Mod 保留完整内容和唯一 Lua。超级版只 `UPDATE GlobalParameters`、追加 Modifier 或追加不重叠 Lua；不要重新 `INSERT` 同名文明/领袖/项目。 |

### 推荐第一版实现取舍

第一版目标是验证玩法闭环，而不是一次做完 UI 和资源美术：

1. 感染者血液先用 `Player Property`，不做真实资源。
2. 所有血液收益、项目扣费、层数上限都放在一个 Gameplay Lua 中。
3. 稳定剂、类固醇、溶血剂都做成最多三层，避免无限叠加。
4. 数值全部放进 `GlobalParameters`，为超级版预留覆盖点。
5. UI 只做文本提示，不做独立血液面板。

这样第一版的风险集中在三处：击杀事件是否稳定、项目完成事件是否稳定、Modifier 效果名称是否选对。其余文明、领袖、建筑、文本和项目定义都可以按数据库常规方式推进。

2. 溶血剂是否允许叠加？如果允许，最高几层？

4. 清理蛮族营地是否额外奖励感染者血液？

## 文明百科风格文本

### 厄尔庇斯协议

厄尔庇斯协议诞生于灾难、隔离与幸存者证词之间。它并不把失控感染视为单纯的灾害，而是把每一次接触都记录为样本，把每一次清剿都整理为数据，把每一滴感染者血液都转化为下一次生还的筹码。它的城市像实验室一样谨慎，像前线据点一样冷峻，也像避难所一样执拗地延续希望。

### 格蕾丝·阿什克罗夫特

格蕾丝·阿什克罗夫特站在隔离线的边缘，在谣言、恐惧与现场证据之间寻找真相。对她而言，蛮族袭击不是普通的边境冲突，而是一次可记录、可追踪、可复盘的感染事件。她的统治让每场战斗都留下样本，每座实验室都承担责任，每一次幸存都成为文明继续前进的理由。

### 罗兹山疗养院

罗兹山疗养院既是医院，也是档案库，更是一座被层层封锁的病理实验室。这里不追求安慰性的答案，只处理危险而具体的事实：稳定伤员，研究异常，封存样本，并把下一次遭遇战的胜算提高一点。它的工作阴冷、缓慢、精确，却正是厄尔庇斯协议得以延续的核心。
